Реализуйте виджет авторизации. В вашем распоряжении форма с полями email и password, которую нужно отправить на сервер. В качестве ответа сервер пришлет данные авторизованного пользователя или ошибку в случае если логин/пароль неправильные.

До отправления формы пользователь должен увидеть только её (форму), все остальные блоки (прелоадер, сообщение об ошибке, блок с данными пользователя) должны быть скрыты. Во время запроса вместо формы пользователь должен увидеть прелоадер. В случае успешной авторизации форма логина и прелоажер должны быть скрыты, а вместо них должна быть отображена краткая информация о пользователе: аватар, полное имя (name lastname), страна и список увлечений (hobbies), увлечения должны быть перечислены через запятую (например one, two, three). В случае ошибки пользователь должен вновь увидеть форму и сообщение об ошибке (полученное с сервера).

Оформление сделайте на ваше усмотрение.

Аттрибуты data-role нужны для проверки правильности выполнения заданий, пожалуйста не изменяйте их, не удаляйте теги с аттрибутом data-role, используйте те, что есть в задании. Вносить изменения в html разметку (добавлять новые теги для оформления, добавлять классы/стили тегам) можно и даже нужно. HTML-комментарии из кода можно удалить.

Задание 1.

Используйте для запроса к API url http://netology-hj-ajax.herokuapp.com/homework/login_json (запрос должен быть отправлен методом POST, данные формы должны быть отправлены в теле запроса). Используйте email test@netology.ru и пароль 12345 для проверки успешной авторизации, при отправлении других значений email/password сервер вернет ошибку. В данном задании сервер присылает ответ в формате XML
